Tic Tac Toe Game Architecture

Game Logic

  • Two human players or One Human Player and One AIBot
  • Game can have multiple rounds and this will be saved in round numbers and number of wins for each player
  • 3 x 3 Grid with clickable buttons that are numbered 1,2,3,4,5,6,7,8,9
  • Two Markers, Marker 1: X, Marker 2: O
  • Players, play turn by turn and game will be over after maximum of 9 turns either in a draw or in a win
  • First player to get 3 of the same mark on a straight line will be the winner before the 9 turn are up according to the following conditions;

Tic Tac Toe Board Coordinates;

1 2 3
4 5 6
7 8 9

Conditions for wins

Straight Line: All three Row wins:

  • 1,2,3
  • 4,5,6
  • 7,8,9

Straight Line: All three Column wins

  • 1,4,7
  • 2,5,8
  • 3,6,9

Straight Line: All three Diagonal wins 1,5,9 3,5,7 Rest of the combination will be a draw ***Main Coding rules*** * Use Object oriented design to create the game and no global variables, store all the the variables inside factories. * Main goal here is to have as little global code as possible. * If only a single instance of something is required (e.g. the gameboard, the displayController etc.) then wrap the factory inside an IIFE (module pattern) so it cannot be reused to create additional instances. * Decide on which Objects to store the relevant logic so code is easily maintainable * First create a working game on the browser console.
